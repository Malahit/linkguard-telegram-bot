# Процесс разработки LinkGuard

> Этот файл — единственный источник правды о том, как мы работаем.
> Обновляется при изменении процесса.

## Базовый цикл (один шаг = один PR)

```
1. Определили задачу  →  2. Зафиксировали в docs  →  3. Разбили на шаг
         ↓
4. Создали ветку feat/... или docs/...
         ↓
5. Сгенерили код — строго в рамках шага
         ↓
6. Открыли PR — с описанием: что сделали + что разблокирует
         ↓
7. Review → правки при необходимости
         ↓
8. Merge → следующий шаг
```

### Правила

- **Не держать в голове, не оставлять в чате** — всё фиксируем в `docs/`
- **Один шаг = одна ответственность** — PR не смешивает два разных изменения
- **Шаг должен иметь чёткий результат** — «добавлен файл X, который делает Y»
- **Код строго в рамках шага** — не берём «заодно» соседние задачи
- **PR описывает**: что сделано + что это разблокирует для следующего шага

### Именование веток

| Тип изменения | Префикс | Пример |
|---|---|---|
| Новая функция | `feat/` | `feat/virustotal-integration` |
| Исправление бага | `fix/` | `fix/bot-crash-on-empty-url` |
| Документация | `docs/` | `docs/dev-process` |
| Рефакторинг | `refactor/` | `refactor/risk-engine-layers` |
| Инфраструктура | `infra/` | `infra/docker-compose` |
