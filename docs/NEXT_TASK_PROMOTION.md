## Задача для агентной модели

### Общая постановка

Разработать и частично автоматизировать стратегию продвижения Telegram-канала «Без страха в сети» (`@bezstrahavseti`) без рекламного бюджета, используя агентные AI-инструменты (Perplexity, автогенерация, парсинг площадок).

### Подзадачи

#### 1. Контент для внешних площадок (агент-генератор)

Создать модуль `ai-promo-post.ts` (по образцу `ai-breakdown.ts`), который:
- принимает тему из свежего поста канала
- генерирует адаптированную версию для Пикабу (длиннее, с личным голосом, с заходом от первого лица)
- генерирует короткую версию-ответ для ВКонтакте (как экспертный комментарий, без прямой рекламы)
- добавляет в конце органическую ссылку на канал

Промпт-база уже есть в `CHANNEL_STYLE.md`.

#### 2. Мониторинг площадок (агент-наблюдатель)

Создать скрипт или cron-задачу, которая:
- раз в сутки парсит или через API ищет свежие посты/вопросы о мошенничестве на ВКонтакте / Пикабу
- формирует список топ-3 тем, где можно дать полезный ответ
- складывает результат в файл или отправляет автору в Telegram как личное сообщение

Технически: можно реализовать через Perplexity API с запросом «свежие кейсы интернет-мошенничества за 24 часа» + парсер ссылок.

#### 3. SEO-оптимизация постов в канале

Проверить текущий `posts-pool.ts` и все AI-генерируемые тексты:
- заголовки должны содержать поисковые фразы («как распознать фишинг», «мошенники в Telegram» и т.д.)
- Telegram индексирует открытые каналы в Google — это бесплатный органический трафик

Задача: составить список из 20–30 SEO-заголовков для будущих постов, соответствующих стилю `CHANNEL_STYLE.md`.

#### 4. Виральный механик «перешли другу»

Добавить в конец каждого поста (в `formatPost` или в промпт AI) призыв к репосту:
- не навязчивый, органичный
- например: «Покажи это другу, который недавно чуть не попался» или «Сохрани — пригодится»

Технически: минимальное изменение в `posts-pool.ts` и в промпте `ai-breakdown.ts`.

#### 5. Черновик-режим для автора

Сейчас посты уходят сразу в канал. Добавить опцию:
- AI-разбор сначала приходит автору в личку как черновик
- автор может одобрить командой `/approve` или отредактировать
- только после этого пост уходит в канал

Технически: новый handler в `telegram-bot.ts` + хранение черновика в `report-store.ts` или новом `draft-store.ts`.
